# Prompt 010: Planejamento de Sprint

## Objetivo
Planejar uma sprint de desenvolvimento com tarefas claras e estimativas.

## Checklist

### 1. Revisão do Estado
- [ ] Ler `PROJECT_STATE.md`
- [ ] Revisar `ROADMAP.md`
- [ ] Verificar `KNOWN_ISSUES.md`
- [ ] Revisar `DECISIONS.md`

### 2. Definição de Objetivos
- [ ] Definir objetivo da sprint
- [ ] Priorizar funcionalidades
- [ ] Estabelecer critérios de aceite
- [ ] Definir prazo

### 3. Criação de Tarefas
- [ ] Decompor funcionalidades em tarefas
- [ ] Estimar cada tarefa
- [ ] Definir dependências
- [ ] Sequenciar execução

### 4. Atualização de Documentos
- [ ] Atualizar `ROADMAP.md`
- [ ] Atualizar `PROJECT_STATE.md`
- [ ] Criar issues no GitHub (se aplicável)

## Template de Sprint

```markdown
# Sprint [Número]

## Objetivo
[Descrição do objetivo]

## Prazo
Início: YYYY-MM-DD
Fim: YYYY-MM-DD

## Tarefas

### Tarefa 1: [Nome]
- **Prioridade:** Alta/Média/Baixa
- **Estimativa:** [X horas/dias]
- **Dependências:** [Nenhuma/Lista]
- **Critérios de Aceite:**
  - [ ] Critério 1
  - [ ] Critério 2

### Tarefa 2: [Nome]
- **Prioridade:** Alta/Média/Baixa
- **Estimativa:** [X horas/dias]
- **Dependências:** [Nenhuma/Lista]
- **Critérios de Aceite:**
  - [ ] Critério 1
  - [ ] Critério 2

## Riscos
| Risco | Impacto | Mitigação |
|-------|---------|-----------|
| [Risco] | Alto/Médio/Baixo | [Ação] |

## Métricas
- Total de tarefas: X
- Estimativa total: Y horas
- Tarefas críticas: Z
```

## Perguntas para o Desenvolvedor

1. Qual o objetivo principal desta sprint?
2. Quais funcionalidades são prioridade?
3. Há dependências externas?
4. Quais são os riscos conhecidos?
5. Qual a estimativa de tempo disponível?

## Saída Esperada

Após completar este prompt:
- Sprint planejada com tarefas
- Estimativas definidas
- Dependências mapeadas
- Riscos identificados
- Documentação atualizada
