# Prompt 020: Durante a Sprint

## Objetivo
Executar as tarefas da sprint seguindo boas práticas e padrões definidos.

## Checklist

### 1. Início do Dia
- [ ] Revisar tarefas do dia
- [ ] Verificar bloqueios
- [ ] Atualizar status em `PROJECT_STATE.md`

### 2. Desenvolvimento
- [ ] Seguir padrões de código
- [ ] Testar enquanto desenvolve
- [ ] Commitar com mensagens semânticas
- [ ] Documentar decisões

### 3. Comunicação
- [ ] Reportar progresso
- [ ] Comunicar bloqueios
- [ ] Pedir ajuda quando necessário

### 4. Fim do Dia
- [ ] Atualizar tarefas
- [ ] Push de commits
- [ ] Documentar lições

## Regras de Desenvolvimento

### Código
1. Seguir padrões do projeto
2. Escrever testes para cada feature
3. Rodar lint antes de commitar
4. Documentar decisões significativas

### Git
1. Usar mensagens semânticas
2. Criar branches para cada tarefa
3. Abrir PR para review
4. Não commitar sem testar

### Comunicação
1. Informar progresso diário
2. Comunicar bloqueios imediatamente
3. Documentar lições aprendidas

## Template de Status Diário

```markdown
## Status - YYYY-MM-DD

### Concluído
- [Tarefa 1]
- [Tarefa 2]

### Em Progresso
- [Tarefa 3]

### Bloqueios
- [Bloqueio 1 - motivo]

### Próximos Passos
- [Tarefa 4]
- [Tarefa 5]

### Lições
- [Lição 1]
```

## Perguntas para o Desenvolvedor

1. Quais tarefas foram concluídas?
2. Há bloqueios?
3. Precisa de ajuda?
4. Alguma lição aprendida?
5. O que será feito amanhã?

## Saída Esperada

Ao longo da sprint:
- Tarefas sendo executadas
- Código testado e documentado
- Progresso comunicado
- Bloqueios resolvidos
- Lições documentadas
