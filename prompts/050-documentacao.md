# Prompt 050: Documentação

## Objetivo
Atualizar e manter documentação do projeto completa e precisa.

## Checklist

### 1. Revisão
- [ ] Identificar documentação desatualizada
- [ ] Verificar links internos
- [ ] Revisar exemplos
- [ ] Verificar completude

### 2. Atualização
- [ ] Atualizar `README.md`
- [ ] Atualizar `ARCHITECTURE.md`
- [ ] Atualizar `DATA_SOURCES.md`
- [ ] Atualizar guias de uso

### 3. Criação
- [ ] Criar nova documentação se necessário
- [ ] Adicionar exemplos
- [ ] Criar tutoriais
- [ ] Documentar APIs

### 4. Validação
- [ ] Verificar formatação
- [ ] Testar exemplos
- [ ] Revisar ortografia
- [ ] Validar links

## O que Documentar

### Sempre
- README.md
- ARCHITECTURE.md
- DECISIONS.md
- CHANGELOG.md

### Quando Muda
- Estrutura de pastas
- Stack tecnológica
- Processos
- Configurações

### Quando Aprende
- Lições aprendidas
- Workarounds
- Dicas importantes

## Template de Documentação

```markdown
# [Título]

## Visão Geral
[Descrição breve]

## Pré-requisitos
- [Requisito 1]
- [Requisito 2]

## Instalação
```bash
# Comandos
```

## Uso
```bash
# Exemplo
```

## Configuração
```javascript
// Configuração
```

## Troubleshooting
### Problema 1
**Solução:** [Solução]

## Referências
- [Link 1]
- [Link 2]
```

## Boas Práticas de Documentação

### 1. Clareza
- Linguagem simples
- Exemplos práticos
- Estrutura lógica

### 2. Completude
- Cobrir todos os cenários
- Incluir pré-requisitos
- Documentar limitações

### 3. Manutenção
- Atualizar regularmente
- Verificar links
- Revisar exemplos

## Perguntas para o Desenvolvedor

1. O que precisa ser documentado?
2. Quem é o público-alvo?
3. Há exemplos disponíveis?
4. Quais são os pontos mais confusos?
5. Há links quebrados?

## Saída Esperada

Após completar este prompt:
- Documentação atualizada
- Exemplos funcionais
- Links verificados
- Completude garantida
