# Prompt 090: Encerramento

## Objetivo
Encerrar sprint ou projeto de forma organizada e documentada.

## Checklist

### 1. Validação
- [ ] Todas as tarefas concluídas
- [ ] Todos os testes passando
- [ ] Documentação atualizada
- [ ] Lições documentadas

### 2. Documentação
- [ ] Atualizar `PROJECT_STATE.md`
- [ ] Atualizar `ROADMAP.md`
- [ ] Atualizar `CHANGELOG.md`
- [ ] Registrar lições aprendidas

### 3. Comunicação
- [ ] Fechar issues resolvidas
- [ ] Notificar stakeholders
- [ ] Compartilhar resultados
- [ ] Agradecer colaboradores

### 4. Limpeza
- [ ] Deletar branches antigas
- [ ] Limpar código morto
- [ ] Atualizar dependências
- [ ] Arquivar sprint

## Template de Encerramento

```markdown

## Resumo
[Resumo do que foi feito]

## Concluído

- [Tarefa 2]
- [Tarefa 3]

## Não Concluído
- [Tarefa 4 - motivo]

## Métricas
| Métrica | Valor |
|---------|-------|
| Tarefas concluídas | X |
| Tarefas pendentes | Y |
| Bugs corrigidos | Z |
| Features adicionadas | W |

## Lições Aprendidas
- [Lição 1]
- [Lição 2]

## Próximos Passos
- [Passo 1]
- [Passo 2]

## Agradecimentos
- [Colaborador 1]
- [Colaborador 2]
```

## Métricas de Sprint

### Coletar
- Tarefas planejadas vs concluídas
- Tempo estimado vs real
- Bugs encontrados vs corrigidos
- Satisfação do time

### Analisar
- O que funcionou?
- O que não funcionou?
- O que melhorar?
- Próximos passos?

## Perguntas para o Desenvolvedor

1. O que foi concluído?
2. O que ficou pendente?
3. Quais lições aprendidas?
4. O que melhorar?
5. Quais são os próximos passos?

## Saída Esperada

Após completar este prompt:
- Sprint/projeto encerrado
- Documentação atualizada
- Lições documentadas
- Próxima sprint planejada
